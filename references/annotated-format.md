# 标注版 Markdown 格式规范（`_annotated.md`）

两阶段排版模式的中间产物。分析完成后产出此文件，排版决策全部写死；第二阶段渲染**只能机械翻译此文件**，不做分析、不推断、不增删。

## 两阶段流程

```
原文.md  →  [阶段1: 分析标注]  →  _annotated.md  →  [阶段2: 机械渲染]  →  HTML
              ↑ LLM 做分析                          ↑ LLM 只做逐元素翻译
              可选人工检查/修正                      可脚本化（确定性）
```

## 文件命名

`{原标题}_annotated.md`，与原稿同目录。

## Frontmatter

```yaml
---
gzh_type: tutorial           # 文章类型（必填）
gzh_theme: moyu-green       # 选定主题标识（必填，对应 theme-index 的英文标识）
gzh_toc:                    # 目录/导读，取最重要的 3 个章节标题（3 项，章节 < 3 时取实际数量）
  - 智能体操控手机的现状
  - 技术实现路径
  - 未来与局限
gzh_tags:                   # 章节英文标签，按 ## 顺序一一对应（可选，无标签的章节写 null）
  - STATUS
  - HOW
  - FUTURE
gzh_author: 金哲             # 作者署名（可选，有则填入；无则省略，尾部签名区用 {{作者名}} 占位）
gzh_bio: 热衷于分享 AI 观察   # 作者一句话简介（可选，配合 gzh_author 使用）
---
```

## 章节标题

章节编号直接写死在 `##` 中：

```markdown
## 01 章节中文标题

## 02 另一个章节

## 03 第三个章节

## ∞ 结语
```

规则：
- 编号严格按 `##` 出现顺序：`01`、`02`、`03`……
- 末章若是结语/总结类，用 `∞`，否则延续数字编号
- `###` 子章节用 `1.1`、`1.2` 编号加在原文标题前
- 章节英文标签**不在标题里出现**，由 frontmatter `gzh_tags` 承载

## 行内标记

分析阶段主动标注关键词——每段 1–3 个短语，用以下标记语法：

| 标记语法 | 语义 | 视觉层级 | 频率 |
|---------|------|---------|------|
| `**文字**` | 加粗（锚点层） | 主色加粗 | 全文 ≤ 5 处 |
| `==文字==` | 高亮 | 渐变背景高亮 | 按需 |
| `++文字++` 或 `<u>文字</u>` | 下划线（核心特色） | 主题下划线 CSS | 高频，每段 1–3 处 |
| `~~文字~~` | 荧光笔（底部半高亮） | 删除线样式 | 按需 |

规则：
- 每个正文段落**至少标 1 个 `++下划线++`**（核心特色），最多 3 个
- `**加粗**` 全文 ≤ 5 处，只在最强锚点（产品名/步骤/CTA/核心金句）使用
- `==高亮==` 优先用于开头引言里的核心词
- `~~荧光笔~~` 用于次强调内容
- 代码块、行内代码内的标记语法**保持原样不处理**
- 原文已有的标记**保留**；分析阶段**新增**的标记叠加在上面

## 引言

- **开头引言**（第一个 `# 标题` 之后、第一个 `##` 之前的 `>` 块）→ 保留原文 `>` 格式，分析阶段可对其中关键词加 `==高亮==`
- **中间引用**（非开头的 `>` 块）→ 保留原文 `>` 格式
- 开头引言有作者/主题落款时，在块末行加 `—— 作者名`（专有署名）或省略（无明确作者）

## 列表

保留原文 Markdown 列表格式（`- ` / `1. `），渲染阶段按主题库映射规则转换。

## 图片 / GIF / 待补素材

保留原文 `![说明](URL)` 格式不变。`![](URL)` 空 alt 不补说明。

待补占位保留原文 `【插入…】` 文字，渲染时走通用库 2c 居中素材占位板块。

## 代码块

保留原文 ` ``` ` 围栏代码块和行内 `` `code` `` 不变。

## 表格

保留原文 Markdown 表格，渲染阶段按主题库映射规则处理。

## 分割线

保留原文 `---` 或 `***`。

## 尾部签名区

在文件末尾、所有章节之后，追加：

```markdown
---

我是 {{作者名}}，{{一句话简介}}。如果你觉得今天这篇有收获，欢迎**点赞、在看、转发**三连，我们下篇见。
```

- 原文末尾已有作者签名段 → 以原文为准，只把作者名/简介填入，保留原文措辞
- `gzh_author` 和 `gzh_bio` 有值 → 替换 `{{作者名}}` 和 `{{简介}}`
- 无值 → 保留占位，交付时提示用户替换

## 标注规则（阶段 1 必遵守）

1. **不改原文文字**：标记语法（`**`、`==`、`++`、`<u>`、`~~`）只能包裹原文已有的词语，不得增删改原文文字
2. **编号只加前缀**：`## 标题` → `## 01 标题`，原文标题文字不动
3. **签名区是追加**：在文件末尾追加，不在原文中间插入
4. **目录只取不改**：frontmatter `gzh_toc` 从 `##` 标题文字中取原文，不重写
5. **半角标点当场转**：标注阶段就把正文半角标点转成全角（代码块内不动）

## 渲染规则（阶段 2 必遵守）

渲染阶段读 `_annotated.md`，规则极简：

- **禁止分析**：不推断文章类型、不选主题、不分章节、不找关键词——这些全在 frontmatter 和标记里
- **逐元素机械翻译**：每个 Markdown 元素按主题库映射规则 → 对应 HTML 组件
- **不增删文字**：`<span leaf="">` 里的文字 = 标注文件里的文字
- **frontmatter 是唯一决策源**：`gzh_type` → 配方表，`gzh_theme` → 主题库，`gzh_toc` → 目录组件，`gzh_tags` → 英文标签
- **章节编号照搬**：`## 01 标题` → 编号是 `01`，`## ∞ 结语` → 编号是 `∞`，不重算
- **标记语义映射**：`**` → 锚点层加粗，`==` → 高亮组件，`++`/`<u>` → 下划线组件，`~~` → 荧光笔组件
